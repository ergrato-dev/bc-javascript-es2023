# 🧩 Proyecto Semanal: Panel CRUD REST

> **🎯 ÚNICO ENTREGABLE**: Este proyecto es el **único entregable obligatorio** para aprobar la semana.

## 🎯 Objetivo

Construir una aplicación CRUD conectada a una API REST que demuestre:

- Operaciones GET, POST, PUT/PATCH y DELETE
- Configuración de headers (incluyendo token de auth)
- Manejo de estados `loading`, `success` y `error`
- Organización modular de funciones

## 📋 Tu Dominio Asignado

**Dominio**: [El instructor te asignará tu dominio al inicio del trimestre]

Debes adaptar nombres de entidades, textos y campos al dominio asignado.

## ⚠️ Nota sobre ejemplos

Los ejemplos de este README usan dominios neutrales de demostración (Museo, Acuario, Planetario) para evitar interferir con la política de dominios únicos del curso.

## 🧪 API de práctica

Usaremos [JSONPlaceholder](https://jsonplaceholder.typicode.com) para practicar el flujo CRUD.

## ✅ Requisitos Funcionales

1. Listar recursos con GET (con paginación básica)
2. Crear recurso con POST
3. Editar recurso con PUT o PATCH
4. Eliminar recurso con DELETE
5. Mostrar estados visuales de cada operación
6. Manejar errores de red y HTTP
