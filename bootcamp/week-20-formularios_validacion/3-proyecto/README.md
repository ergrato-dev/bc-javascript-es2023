# 🏛️ Proyecto Semanal: Sistema de Registro y Login con Validación

> **🎯 ÚNICO ENTREGABLE**: Este proyecto es el único entregable obligatorio para aprobar la semana.

## 🎯 Objetivo

Implementar un flujo de registro/login aplicando:

- Captura de datos con `FormData`
- Validación HTML5 y Constraint Validation API
- Reglas personalizadas de negocio
- Submit handling robusto con feedback claro

## 📋 Tu dominio asignado

Dominio: asignado por el instructor (ej. Biblioteca, Farmacia, Gimnasio, etc.).

Adapta nombres de campos y reglas del formulario a tu dominio.

## ✅ Requisitos funcionales

1. Formulario de registro con validación completa
2. Formulario de login con validación mínima
3. Mensajes de error por campo
4. Estado de envío (cargando, éxito, error)
5. Prevención de dobles envíos
6. Persistencia temporal opcional para mejorar UX

## 🧠 Requisitos técnicos

1. Usar `const`/`let`, arrow functions y destructuring
2. Capturar datos con `FormData`
3. Integrar `checkValidity`, `reportValidity`, `setCustomValidity`
4. Separar validaciones en funciones reutilizables
5. Mantener nomenclatura técnica en inglés y comentarios en español

## 📦 Entregables

1. Código funcional adaptado al dominio
2. Validaciones demostrables en ejecución
3. Evidencia visual (capturas o video corto)

## 🧪 Checklist final

- [ ] Registro validado con reglas nativas y personalizadas
- [ ] Login validado con reglas mínimas claras
- [ ] Submit handling sin dobles envíos
- [ ] Mensajería útil para el usuario
- [ ] Adaptación coherente al dominio asignado
