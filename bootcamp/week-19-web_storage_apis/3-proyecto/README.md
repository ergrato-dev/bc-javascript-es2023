# 🏛️ Proyecto Semanal: Aplicación con Persistencia de Datos Local

> **🎯 ÚNICO ENTREGABLE**: Este proyecto es el único entregable obligatorio para aprobar la semana.

## 🎯 Objetivo

Construir una aplicación con persistencia local robusta aplicando:

- `localStorage` y `sessionStorage`
- Serialización segura con JSON
- Almacenamiento histórico básico con IndexedDB
- Manejo de errores de cuota y fallback

## 📋 Tu dominio asignado

Dominio: asignado por el instructor (ej. Biblioteca, Farmacia, Gimnasio, etc.).

Adapta nombres, campos y flujos de la app a tu dominio.

## ✅ Requisitos funcionales

1. Renderizar listado inicial de elementos del dominio
2. Crear nuevos elementos desde formulario
3. Persistir estado principal en localStorage
4. Guardar estado de vista temporal en sessionStorage
5. Registrar historial básico en IndexedDB (simulado o real)
6. Mostrar mensaje de error y fallback si falla persistencia

## 🧠 Requisitos técnicos

1. Usar `const`/`let`, arrow functions y destructuring
2. Implementar funciones utilitarias de storage
3. Manejar parseo seguro con fallback
4. Documentar decisiones clave con comentarios breves en español
5. Mantener nomenclatura técnica en inglés

## 📦 Entregables

1. Código funcional adaptado al dominio asignado
2. Persistencia local comprobable
3. Capturas o evidencia de funcionamiento

- [ ] Manejo de errores al guardar
- [ ] Uso coherente de local/session storage
